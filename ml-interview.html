Model comparison

Know which model you would choose and why.

Categorical data

Tree: needs encoding.

Forest: needs encoding; high-cardinality can mislead importance.

XGBoost: strong, but preprocessing matters.

CatBoost: usually best default because ordered target statistics are built in.

Overfitting

Tree: high risk unless constrained or pruned.

Forest: reduces variance through averaging.

XGBoost: controls with learning rate, depth, subsampling, regularization.

CatBoost: ordered boosting reduces target leakage and prediction shift.

Interpretability

Tree: easiest to explain, if shallow.

Forest: harder; use permutation importance or SHAP.

XGBoost: strong but needs post-hoc explanation.

CatBoost: similar to other boosting ensembles.

Training speed

Tree: fast and simple.

Forest: parallelizable across trees.

XGBoost: optimized for large tabular tasks.

CatBoost: categorical processing can cost more, but saves preprocessing time.

Interview script

Use this sequence when asked to compare models.

  1. Start from the data: tabular, categorical, missing values, size, latency, and metric.
  2. Name the bias-variance tradeoff: tree overfits, forest averages, boosting reduces bias sequentially.
  3. Explain leakage risk for categorical target encodings and why CatBoost handles it carefully.
  4. Mention regularization knobs before hyperparameter names become a list.
  5. End with validation: time split, group split, OOB, or cross-validation depending on the task.
