Bagging ensemble

A random forest makes unstable trees useful by averaging them.

Each tree sees a bootstrap sample and a random subset of features at each split. The individual trees stay noisy, but their average has much lower variance.

Interview answer Average decorrelated trees.

Bagging reduces variance. Feature subsampling reduces correlation between trees.

Core mechanics

What randomness is injected?

Bootstrap rows

Each tree trains on a sample drawn with replacement. Some rows repeat, and some are left out for that tree.

Random features

At every split, the tree considers only a subset of features. This prevents one dominant feature from making all trees too similar.

Aggregation

Classification averages class probabilities or votes. Regression averages numeric predictions.

OOB score

For each row, predict using only trees where that row was not in the bootstrap sample. This gives a built-in validation estimate.

Variance story

The key sentence is about correlation.

Averaging independent models Var(mean) = Var(tree) / T

This is the ideal case, but forest trees are not independent.

Correlated trees lower correlation => stronger variance reduction

Feature subsampling exists to reduce correlation, not just to make training faster.

Bias forest bias ≈ deep tree bias

Random forests mainly reduce variance. They do not build additive corrections like boosting.

Drills

Interview answers that separate forest from boosting.

Why not train identical trees on the full dataset?

Answer

If trees are identical or highly correlated, averaging gives little benefit. Bootstrap sampling and random features create diversity, which reduces variance.

Does a random forest overfit as trees are added?

Answer

Usually adding more trees stabilizes the ensemble instead of overfitting. Overfitting is controlled more by tree depth, leaf size, feature leakage, and data noise.

Traps

Things interviewers often probe.

OOB is not magic.

It estimates validation performance from unused bootstrap rows, but it does not replace the right time or group split.

Impurity importance is biased.

Prefer permutation importance when cardinality or correlated features can distort the story.

Forest is not sequential.

Trees are trained independently, so the model is easy to parallelize and differs fundamentally from boosting.
