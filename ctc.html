Mental model

CTC is path summation, not forced segmentation.

A network emits a symbol distribution at every frame. CTC asks: "How much total probability did the network assign to every frame-by-frame path that collapses to my target?"

1. Each frame chooses a raw symbol.

The raw path can contain blanks and repeated symbols because timing is unknown. Blank is not a printed space; it means no new label now.

2. Collapse removes timing.

First merge consecutive duplicates, then delete blanks. Many raw paths can become the same target.

3. DP stores prefix mass.

Each cell alpha[t,s] is the probability of being at extended-target state s after frame t.

4. Repeats need a blank gap.

For targets like AA, jumping from A to A is blocked, so the path must pass through blank.

5. Beta is future mass.

It starts at the final states and asks how much probability can still finish the target.

6. Alpha times beta gives credit.

The product shows how much successful target probability passes through a frame-state cell.

Deep intuition

Imagine pouring probability mass through a left-to-right lattice. Columns are states in the extended target: blank, first label, blank, second label, and so on. At every new frame, mass can stay, advance by one state, or sometimes skip over a blank. The emission probability at that frame acts like a gate: if the network likes the symbol for this column, more mass passes through.

The loss is small when a lot of mass reaches either final column. It does not matter which exact alignment got there. That is why CTC works for speech, handwriting, OCR, and other sequence tasks where labels are known but frame-level boundaries are not.

The recurrences

alpha[t,s] = p_t(label_s) * (stay + advance + skip) beta[t,s] = sum p_{t+1}(next_label) * beta[t+1,next_s]

Alpha moves probability forward from the past. Beta moves credit backward from the future. The same stay, advance, and skip rules are used in reverse; the skip is blocked for blanks and repeated labels.
