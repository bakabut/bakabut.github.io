Categorical boosting

CatBoost is gradient boosting built to avoid categorical leakage.

The interview point is not just “it handles categories.” The point is that naive target encoding leaks target information, and CatBoost uses ordered statistics and ordered boosting to reduce that prediction shift.

Interview answer Target stats without peeking.

For a row, categorical encodings are computed from earlier rows in a permutation.

Core mechanics

Why CatBoost exists.

Naive target encoding leaks

If a category value is encoded using the target of the same row, the feature contains direct label information during training.

Ordered target statistics

CatBoost samples permutations and encodes each row using only previous rows, plus priors for smoothing rare categories.

Ordered boosting

The model tries to reduce prediction shift: training-time residuals should look closer to what happens at inference.

Symmetric trees

CatBoost commonly uses oblivious trees where all nodes at the same depth use the same split, improving speed and regularity.

Formulas

Target statistics in one line.

Smoothed category mean TS = (sum previous y + prior * a) / (count previous + a)

The prior prevents rare categories from becoming overconfident.

No self-target row i uses rows j where permutation_rank(j) < permutation_rank(i)

This is the core leakage-prevention sentence.

Boosting frame F_t(x) = F_{t-1}(x) + η f_t(x)

CatBoost is still gradient boosting; categorical handling is the differentiator.

Drills

Questions likely in a Yandex-flavored tabular ML interview.

Why is naive target encoding dangerous?

Answer

The encoded feature can include the current row’s label. Training loss becomes too optimistic, and validation or production performance drops.

When would you choose CatBoost?

Answer

For tabular data with important categorical features, especially high-cardinality categories, limited preprocessing time, and leakage-sensitive validation.

Traps

What not to say.

“CatBoost just one-hot encodes.”

It uses target statistics and categorical combinations; one-hot is only part of the broader handling.

“Target encoding is always leakage.”

It is dangerous when done naively. Ordered or cross-fitted encodings are designed to avoid peeking.

“CatBoost removes validation needs.”

You still need validation matching production, especially for time, user, or group leakage.
